# 迭代 003

## 当前理解

内容库是可整体替换的只读数据库。列表不能读取 `extra_json`，详情才解析 JSON，所有内容查询必须在 IO 调度器中执行。

## 实现方案

在只读数据库对象中提供类型数量、摘要、详情、分类、别名和批量 ID 查询，并以 Repository 隔离 UI。详情只按 `extra_json` 的对象结构解析，未知字段不影响读取。

## 本次变更

- 增加标准实体摘要、详情、翻译状态和类型数量模型。
- 增加按类型排序的列表、按 ID 批量读取、详情、别名和分类查询。
- 保持 `SQLiteDatabase.OPEN_READONLY` 与 `PRAGMA query_only`，所有查询使用绑定参数。
