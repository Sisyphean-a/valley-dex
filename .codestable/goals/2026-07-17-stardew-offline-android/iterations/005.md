# Iteration 005

## 当前理解

上一版详情页只输出整段结构化 JSON，不能满足离线查询中按类型浏览和关系跳转的需求。

## 实现方案

在数据层解析稳定的 `officialDerived` 和明确的根字段；界面只接收字段、关系和已解析目标。关联 ID 依次尝试限定 ID、游戏物品限定 ID 与兼容类型，找不到时保留原始值。

## 本次变更

- 设计并实现通用详情头部、类型字段、关系、笔记和原始数据展开。
- 覆盖物品、作物、鱼类、村民、配方、装备、商店和掉落；其他类型按已知字段与原始数据降级。
- 增加本地图片路径边界检查、关系跳转和未解析关联提示。
- 增加格式化、详情解析和关系候选的单元测试。

## 验证证据

`./gradlew.bat :app:testDebugUnitTest :app:lintDebug` 成功。测试覆盖作物、配方、商店、季节、时间、概率和关系 ID 候选。
