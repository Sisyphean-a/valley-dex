# 界面设计

## 设计目标

这是一个高频查询工具，不是游戏复刻或营销页面。内容优先级依次为名称、可检索性、关系和用户记录；装饰只保留轻微的像素质感。

## 视觉语言

- 使用 Material 3 的紧凑密度与静态主题色，不使用动态取色。
- 主色为森林绿，强调色为琥珀黄，错误色为砖红；浅色背景为中性白灰，深色背景为墨绿灰。
- 列表、筛选和关系使用 4dp 圆角与细边框；不堆叠卡片，不使用渐变或装饰性插画。
- 仅在图标容器和类型标签使用像素感方角，保持文字和触控操作的原生可读性。

## 信息架构

底部固定四个主入口：首页、搜索、收藏、更多。更多页收纳历史、数据管理、设置、关于和许可证；详情、类型列表与数据导入为层级页面。

```text
启动 / 导入
  -> 首页 -> 类型列表 -> 实体详情
  -> 搜索 -> 实体详情
  -> 收藏 -> 实体详情
  -> 更多 -> 历史 / 数据管理 / 设置 / 关于
```

## 关键页面

### 数据导入

首屏直接说明当前未载入数据，并提供一个全宽的“选择数据包”命令。导入过程展示阶段、进度和可展开的错误原因；不显示虚构分类或条目。

### 首页

顶部是搜索入口，随后按真实数据库数量显示常用类型网格、最近浏览和当前数据版本。所有分类按钮显示图标、名称和数量，零数量类型不渲染。

### 搜索与列表

搜索框始终在页面顶部，结果用固定高度的紧凑条目呈现：本地图片、中文名、可选英文名、类型/分类和命中原因。筛选用可滚动的 Chip 行，空状态明确说明下一步。

### 实体详情

顶部按图片、中文名、英文名、类型和分类排列。下面依次放入类型字段、关系、个人笔记和高级原始数据。每个分组只在有内容时显示，关系条目可直接跳转；无法解析的 ID 保留原值并标注缺失。

## 可访问性

所有可点击目标不小于 48dp，图标有语义说明，状态同时用文字表达。布局可在字体放大、横屏与软键盘显示时滚动，不依赖颜色区分状态。
